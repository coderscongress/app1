Mastering Software Design Patterns In JavaScript

Software design patterns are essential tools in the arsenal of any skilled software developer. They provide proven solutions to common problems that arise during software development, promoting code reusability, maintainability, and scalability. In this article, we'll explore some popular design patterns and illustrate their application with relevant code snippets.

Introduction to Design Patterns

Design patterns are general solutions to recurring problems that software developers encounter. They encapsulate best practices and provide a common vocabulary for communicating complex design concepts. In JavaScript, design patterns can be particularly helpful due to the language's dynamic nature.

Creational Patterns

Singleton Pattern

The Singleton Pattern ensures that a class has only one instance and provides a global point of access to that instance.


JavaScript Code Snippet

 
const Singleton = (function () {
  let instance;

  function createInstance() {
    // Singleton logic here
    return {};
  }

  return {
    getInstance: function () {
      if (!instance) {
        instance = createInstance();
      }
      return instance;
    },
  };
})();

const instance1 = Singleton.getInstance();
const instance2 = Singleton.getInstance();

console.log(instance1 === instance2); 
          // Output: true

Factory Pattern

The Factory Pattern provides an interface for creating objects in a super class, but allows subclasses to alter the type of objects that will be created.


JavaScript Code Snippet

 
class Product {
  constructor(name) {
    this.name = name;
  }
}

class ConcreteProductA extends Product {
  constructor() {
    super('Product A');
  }
}

class ConcreteProductB extends Product {
  constructor() {
    super('Product B');
  }
}

class ProductFactory {
  createProduct(type) {
    switch (type) {
      case 'A':
        return new ConcreteProductA();
      case 'B':
        return new ConcreteProductB();
      default:
        throw new Error('Invalid product type');
    }
  }
}

const factory = new ProductFactory();
const productA = factory.createProduct('A');
const productB = factory.createProduct('B');

 Structural Patterns

Module Pattern

The Module Pattern allows encapsulation of a group of related functionality into a single unit, preventing global scope pollution.



JavaScript Code Snippet

 
const MyModule = (function () {
  let privateVar = 'This is private';

  function privateFunction() {
    console.log(privateVar);
  }

  return {
    publicFunction: function () {
      privateFunction();
    },
  };
})();

MyModule.publicFunction(); // Output: This is private

Decorator Pattern

The Decorator Pattern lets you attach additional behaviors to objects dynamically without modifying their code.

JavaScript Code Snippet

 
class Coffee {
  cost() {
    return 5;
  }
}

class MilkDecorator {
  constructor(coffee) {
    this.coffee = coffee;
  }

  cost() {
    return this.coffee.cost() + 2;
  }
}

class SugarDecorator {
  constructor(coffee) {
    this.coffee = coffee;
  }

  cost() {
    return this.coffee.cost() + 1;
  }
}

let myCoffee = new Coffee();
myCoffee = new MilkDecorator(myCoffee);
myCoffee = new SugarDecorator(myCoffee);

console.log(myCoffee.cost()); // Output: 8

Behavioral Patterns

Observer Pattern

The Observer Pattern defines a one-to-many relationship between objects. When one object changes state, all its dependents are notified and updated automatically.


JavaScript Code Snippet

 
class Subject {
  constructor() {
    this.observers = [];
  }

  addObserver(observer) {
    this.observers.push(observer);
  }

  notify(message) {
    this.observers.forEach((observer) => {
      observer.update(message);
    });
  }
}

class Observer {
  update(message) {
    console.log(`Received message: ${message}`);
  }
}

const subject = new Subject();
const observer1 = new Observer();
const observer2 = new Observer();

subject.addObserver(observer1);
subject.addObserver(observer2);

subject.notify('Hello, observers!');

Strategy Pattern

The Strategy Pattern defines a family of interchangeable algorithms and lets them be easily swapped out.


JavaScript Code Snippet

 
class PaymentStrategy {

  pay(amount) {

    throw new Error('This method should be overridden');

  }

}



class CreditCardPayment extends PaymentStrategy {

  pay(amount) {

    console.log(`Paid $${amount} with credit card`);

  }

}



class PayPalPayment extends PaymentStrategy {

  pay(amount) {

    console.log(`Paid $${amount} with PayPal`);

  }

}



class ShoppingCart {

  constructor(paymentStrategy) {

    this.paymentStrategy = paymentStrategy;

  }



  checkout(amount) {

    this.paymentStrategy.pay(amount);

  }

}



const cart = new ShoppingCart(new CreditCardPayment());

cart.checkout(100); // Output: Paid $100 with credit card
